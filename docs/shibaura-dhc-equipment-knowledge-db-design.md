# 設備ナレッジDB兼 季節切替支援マップ 設計ドキュメント

対象：芝浦DHC 第2プラント（地下1階 機器配置図ベース）

## 1. 背景・課題

- 季節切替（冷房⇔暖房）の運転・操作が熟練者の経験に依存している（属人化）
- 新入社員が、系統構成・機器の設置位置・操作順序を理解できない
- 平面図／LS（リミットスイッチ）図／I/O表が個別に存在し、相互に連携していない
- 「どこに何があり、何をどう操作すればよいか」が一連の流れとして繋がっていない

## 2. ゴール

新人でも単独で理解・判断できる「設備ナレッジDB兼 季節切替支援マップ」を構築する。

単なる図面集ではなく、以下を統合した情報基盤とする：

- 平面図（位置情報）
- 機器情報（定格・写真・用途）
- 系統情報（冷房・暖房フロー）
- LS情報（状態監視）
- 操作手順（季節切替）
- I/O情報（中央監視連携）

## 3. 成果物

### 3-1. 冷房運転系統図（平面ベース）
- 冷房運転時に稼働する機器のみを強調表示
- 稼働系統（ポンプ・熱源機・弁）を明示
- 非稼働系統はグレーアウトし、視認性を確保

### 3-2. 暖房運転系統図（平面ベース）
- 暖房運転時の系統を同様の表現で可視化
- 冷房図との対比により、季節間の差分を直感的に把握できる構成とする

### 3-3. 季節切替手順付き平面図（最重要成果物）
- 平面図上に番号付きの操作手順を配置し、操作順序をそのまま図面上でトレース可能にする

操作手順の例：
1. WV-12 閉
2. WV-15 開
3. LS-745 確認
4. P-2 起動

→ 図面上に①②③④の順序でマーキングし、現場で迷わず辿れる導線とする

## 4. データ構造（カード単位）

各機器・LS・弁は、相互にリンク可能な独立した情報単位（カード）として管理する。

### 4-1. 機器カード
| 項目 | 内容 |
|---|---|
| 写真 | 現物写真 |
| 仕様 | 定格・型式等 |
| 設置位置 | 平面図上の座標／エリア |
| 系統 | 所属する冷房／暖房系統 |
| 関連LS | 紐づくLSカードへのリンク |
| 関連手順 | 紐づく操作手順へのリンク |

### 4-2. LSカード
| 項目 | 内容 |
|---|---|
| 位置 | 平面図上の設置位置 |
| 対象弁 | 監視・連動する弁 |
| 中央監視点 | 中央監視システム上のポイント番号 |
| 状態定義 | ON/OFF・開/閉等の状態の意味 |

### 4-3. 弁カード（WV等）
| 項目 | 内容 |
|---|---|
| 開閉状態 | 現在の開閉状態 |
| 操作位置 | 操作箇所（現地／中央） |
| 系統 | 所属する冷房／暖房系統 |
| 関連LS | 紐づくLSカードへのリンク |

## 5. ナビゲーション設計（最終イメージ）

一枚の図面で完結させるのではなく、以下のように情報を辿れる統合型構成とする。

```
平面図
  └─ クリック → 機器情報
       └─ 系統図
            └─ 操作手順
                 └─ LS状態
                      └─ I/O情報
```

利用者は平面図から出発し、必要な情報（機器仕様・系統・手順・LS状態・I/O）へ段階的にドリルダウンできる。

## 6. 設計思想

- 属人知識の排除：ベテランの頭の中にある系統理解を、外部から参照可能な形に外部化する
- 図面と手順の統合：図面（位置）と操作（手順）を切り離さず、一体として提示する
- 現場での迷い削減：「どこで」「何を」「どの順で」操作するかを、見れば分かる状態にする
- 「探す」作業の排除：図面・機器情報・手順・LS・I/Oを横断検索／リンクで直結する
- 長期運用可能な基盤：単発の資料ではなく、機器更新・人員交代後も使い続けられるナレッジ基盤として設計する

## 7. 今後の拡張方針

- 4章のカード構造（機器／LS／弁）をベースに、データを段階的に登録・蓄積していく
- まずは冷房系統図・暖房系統図・季節切替手順図の3点（3章）を優先的に整備する
- カード間のリンク関係が整った段階で、5章のドリルダウン型ナビゲーションを実装する
